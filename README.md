# Портфолио-сайт в стиле киберпанк

Стильный портфолио-сайт с эффектами глитча, неоновыми элементами и интерактивными кнопками в современном киберпанк-стиле.

## Особенности

- Анимированный матричный фон
- Интерактивные киберпанк-эффекты при наведении на элементы
- 10+ видов глитч-эффектов для кнопок
- Форма обратной связи с серверной обработкой
- Полностью адаптивный дизайн
- Поддержка бэкенда на Flask для обработки контактных форм

## Установка и локальный запуск

### Статический сайт (только фронтенд)

Просто откройте `index.html` в браузере или используйте любой HTTP-сервер.

```bash
# Пример с использованием Python
python -m http.server
```

### С поддержкой бэкенда (Flask)

1. Установите зависимости:

```bash
pip install -r server/requirements.txt
```

2. Запустите Flask-сервер:

```bash
python server/app.py
```

3. Откройте `http://localhost:5000` в браузере.

## Деплой

### На GitHub Pages (статический сайт)

1. Создайте репозиторий на GitHub
2. Загрузите файлы сайта (`index.html`, `styles.css`, `script.js`)
3. Включите GitHub Pages в настройках репозитория

### На Render (с Flask)

1. Создайте новый Web Service на [Render](https://render.com)
2. Укажите репозиторий с кодом
3. Выберите тип "Web Service"
4. Укажите команду для запуска: `gunicorn app:app --chdir server`
5. Укажите Python 3.9 или выше

### На Heroku (с Flask)

1. Установите [Heroku CLI](https://devcenter.heroku.com/articles/heroku-cli)
2. Создайте новое приложение:

```bash
heroku create your-portfolio-name
```

3. Загрузите код:

```bash
git push heroku main
```

## Настройка отправки электронной почты

Для активации отправки писем с формы контактов:

1. Откройте `server/app.py`
2. Обновите EMAIL_CONFIG с вашими данными
3. Для Gmail создайте [пароль приложения](https://support.google.com/accounts/answer/185833)
4. Раскомментируйте строку с вызовом функции `send_email()` в функции `submit_contact()`

## Безопасность

Для обеспечения безопасности приложения выполните следующие шаги:

1. **Настройте переменные окружения**:
   ```bash
   cp server/.env.example server/.env
   ```
   Измените значения в `.env` файле на ваши собственные

2. **Создайте надежный секретный ключ**:
   ```bash
   python -c "import secrets; print(secrets.token_hex(16))"
   ```
   Полученное значение поместите в `.env` файл для параметра SECRET_KEY

3. **Включите HTTPS** при размещении на продакшн-сервере для защиты CSRF-токенов и передаваемых данных

4. **Защита от спама**:
   - Система имеет базовую защиту от спама с фильтрацией подозрительных сообщений
   - При необходимости добавьте [reCAPTCHA](https://developers.google.com/recaptcha), раскомментировав соответствующий код

5. **Защита от XSS-атак**:
   - Все входящие данные очищаются с помощью библиотеки `bleach`
   - Установлена защита от межсайтовой подделки запросов (CSRF)
